# Sky Field Equipment System — HTML + Supabase

النظام يعمل من ملف `index.html` واحد على GitHub Pages، بينما توفر Supabase تسجيل الدخول وقاعدة البيانات وتخزين شهادات المعايرة. لا يحتاج إلى Node.js أو MongoDB أو Google Apps Script.

## العربية والإنجليزية

- استخدم زر **English / العربية** في شاشة الدخول أو الشريط العلوي لتبديل لغة الواجهة.
- يتغير اتجاه الصفحة تلقائيًا بين RTL للعربية وLTR للإنجليزية، ويُحفظ اختيار كل مستخدم في المتصفح.
- الترجمة تغيّر النص المعروض فقط؛ قيم الحالات والمواقع المحفوظة في Supabase لا تتغير.

## الإعداد لأول مرة

1. أنشئ مشروعًا في Supabase.
2. افتح **SQL Editor** وشغّل ملف `supabase-schema.sql` كاملًا مرة واحدة.
3. من **Authentication > Users** أنشئ أول مستخدم بالبريد الإلكتروني وكلمة المرور.
4. انسخ UUID لهذا المستخدم، ثم نفّذ في SQL Editor:

```sql
update public.sf_profiles
set role = 'admin'
where id = 'ضع UUID المستخدم هنا';
```

5. افتح `index.html` وسجّل الدخول بالبريد وكلمة المرور التي أنشأتها.

إذا كان هذا أول مشروع ويوجد مستخدم Auth واحد فقط، يمكنك تشغيل ملف
افتح `enable-user-admin-by-email.sql`، واستبدل `PUT_EMAIL_HERE` ببريد الحساب مرة واحدة، ثم شغّل الملف كاملًا. يعمل السكربت حتى عند وجود أكثر من مستخدم. بعد نجاحه سجّل الخروج من النظام ثم ادخل مرة أخرى؛ سيظهر الدور **مدير** وتظهر أزرار الإضافة والتعديل.

النسخة الحالية مربوطة مسبقًا بمشروع Supabase الخاص بـ Sky Field. زر **Supabase** في أعلى النظام متاح لتغيير الاتصال مستقبلًا عند الحاجة.

إذا كانت النسخة القديمة استُخدمت من المتصفح نفسه وكانت Supabase فارغة، سيعرض النظام نقل الأجهزة والسجلات والمهندسين المحفوظين محليًا إلى Supabase تلقائيًا. احتفظ بنسخة من Google Sheets قبل الانتقال كإجراء احتياطي.

## النشر على GitHub Pages

بعد تنفيذ ملف SQL وإنشاء المستخدم، يكفي رفع `index.html` إلى مستودع GitHub Pages؛ رابط المشروع والمفتاح العام موجودان داخل النسخة الحالية، لذلك لا يحتاج كل جهاز إلى ضبط الاتصال يدويًا.

## الأدوار والصلاحيات

- `admin`: إدارة كاملة، ويمكنه تغيير أدوار المستخدمين من Supabase.
- `storekeeper`: إضافة وتعديل وحذف الأجهزة والحركات والمهندسين والمواقع.
- `viewer`: عرض البيانات فقط.

أي مستخدم جديد يحصل تلقائيًا على دور `viewer`. لتغيير الدور:

```sql
update public.sf_profiles set role = 'storekeeper' where id = 'USER_UUID';
```

## المواصفات والمعايرة

- ملف `Sky Field - Database.xlsx` يحتوي الماركات والمواصفات الكاملة للأجهزة الـ39، وملف الاستيراد المرفق ينقلها إلى Supabase.
- لتحديث الماركات والمواصفات فقط دون تغيير مواقع الأجهزة أو العهدة أو المعايرة، شغّل `SkyField_Brands_Specs_Update.sql` في Supabase SQL Editor.
- للتعديل اليدوي: افتح **إدارة الأجهزة** ثم **تعديل**، وأدخل الماركة والمواصفات.
- لتفعيل المعايرة: فعّل **هذا الجهاز يتطلب معايرة دورية**، ثم أدخل رقم الشهادة وتاريخ آخر معايرة وتاريخ الانتهاء والمعمل.
- يمكن رفع شهادة بصيغة PDF أو صورة بحد أقصى 10MB؛ تُحفظ داخل Supabase Storage.
- لتعبئة المعايرة للأجهزة الثمانية التي وصلت شهاداتها، شغّل `SkyField_Calibration_Metadata_Update.sql` ثم ارفع ملف PDF المطابق لكل جهاز من حزمة `SkyField_Calibration_Certificates.zip` عبر شاشة تعديل الجهاز.
- للتحديث الجماعي: يمكن استيراد ملف **Sky Field - Database.xlsx** مباشرة؛ سيقرأ النظام ورقة `المخزون`. أو نزّل **قالب المواصفات والمعايرة**، أكمل الأعمدة، ثم استخدم **استيراد التحديثات**. رفع ملف الشهادة نفسه يتم يدويًا لكل جهاز، بينما يمكن استيراد رابط الشهادة من القالب.
- عند نقل جهاز إلى مستلم، يتغير موقعه في المخزون إلى موقع الحركة المختار. وعند إعادته إلى مدير المخزن يرجع الموقع تلقائيًا إلى `المستودع`.

## النقل بين الأشخاص

- عند اختيار جهاز، يلتقط النظام المُسلِّم تلقائيًا من اسم الحائز الحالي المسجل على الجهاز؛ لا توجد خانة يدوية للمُسلِّم.
- يختار المستخدم المستلم الجديد فقط، وبعد حفظ الحركة تنتقل عهدة الجهاز إليه.
- يظهر اسما المُسلِّم والمستلم في سجل الحركات، وتقرير الطباعة، وتصدير Excel.
- قبل نشر الإصدار 9 على قاعدة موجودة، شغّل `supabase-v9-transfer-update.sql` مرة واحدة في Supabase SQL Editor.

## الأمان

- استخدم داخل HTML مفتاح **Publishable/Anon** فقط.
- لا تضع `service_role key` داخل HTML مطلقًا.
- جميع الجداول والملفات محمية بسياسات Row Level Security.
- شهادات المعايرة محفوظة في bucket خاص، ويتم فتحها بروابط مؤقتة مدتها دقيقتان.
